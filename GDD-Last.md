## 0. 三版 GDD 整合结论

本最终版以三份文档的优势互补为基础，形成一个可直接进入 MVP 开发与微信小游戏上线验证的统一方案。

### 0.1 采用内容

- 《GDD.md》提供核心玩法骨架：水晶碎片、光路构筑、实时预览、Perfect Flow、Near Miss、光能经济与基础关卡框架。
- 《GDD 2.md》提供微信小游戏产品化方案：FTUE、新手强引导、异步社交、残局分享、互助充能、广告与轻度内购点位。
- 《GDD 3.md》提供实际开发落地方案：Cocos + TypeScript、2.5D 逻辑网格、Runner 状态机、方向映射表、覆盖机制、子弹时间、零美术实现路线。

### 0.2 最终取舍

- 最终玩法定位采用《GDD 3.md》的“逆向跑酷 + 动态路径规划”，比单纯静态光路解谜更适合微信小游戏短局、高频、强刺激的生态。
- 最终视觉定位保留《GDD.md》和《GDD 2.md》的“等距水晶低多边形”，但实现方式采用《GDD 3.md》的零切图、纯引擎动态渲染。
- 最终商业化采用《GDD 2.md》的微信生态方案，并用《GDD 3.md》的第四手牌槽、子弹时间补能、主题色解锁作为低成本可落地广告点。
- 最终核心技术以方向映射表和网格状态机为主，不做复杂物理模拟，确保 Cocos 微信小游戏端性能稳定。

### 0.3 本轮优化结论

针对“玩法有吸引力、易学易懂、进阶有挑战、容易微信裂变、可以广告变现”的目标，v1.1 做以下强化：

- 强化第一分钟吸引力：新增“先爽后难”的 3 步体验，玩家首次拖拽后立刻看到光流爆发、星级反馈和可分享光轨。
- 降低理解门槛：把规则收敛为“接路、转弯、别掉下去”三句话，所有复杂机制都延后出现。
- 增强进阶深度：新增关卡挑战矩阵，将难度拆成时间、空间、手牌、机关、分流、颜色 6 个维度，避免只靠加速制造难度。
- 增强裂变闭环：新增“求助、炫耀、挑战、互惠”四类分享触发器，每次分享都有明确回流目标。
- 增强广告转化：广告点从“泛奖励”调整为“痛点时刻奖励”，并加入频控、A/B 指标和非强迫原则。

## 1. 项目概述

### 1.1 游戏名称

《浮岛浮光 Float & Flow》

### 1.2 游戏类型

策略构筑 / 动态路径规划 / 逆向跑酷 / 轻解谜 / 微信休闲小游戏

### 1.3 一句话定义

玩家在等距水晶浮岛网格中，拖拽随机发放的水晶道路碎片，为自动滑行的发光水晶块实时铺路、转向、分流和救场，引导其抵达终点或在无尽模式中尽可能延长光轨。

### 1.4 核心体验

- 单指拖拽，3 秒上手。
- 路径即时预览，放置前即可看到光流走向。
- Runner 自动前进，玩家在压力下进行前方道路构筑。
- Near Miss 制造“差一点就过”的强烈救场欲望。
- Perfect Flow 提供一气呵成、秩序感和视觉爽感。

### 1.4.1 玩法吸引力设计支柱

| 支柱 | 玩家感受 | 设计落点 |
| --- | --- | --- |
| 一拖就懂 | “我知道我要做什么” | 只要求玩家把路块拖到断点前方，系统自动预览成败 |
| 一通就爽 | “光一下子流过去了” | 成功连通时播放连续光轨、粒子爆发、音高上扬和轻震 |
| 差点想救 | “就差一点，不想放弃” | Near Miss 聚焦终点、显示差距，并给出救场选项 |
| 越玩越聪明 | “我能优化路线” | 三星、Perfect、最短路径、分流连锁形成进阶目标 |
| 分享有理由 | “这个残局值得发给朋友” | 失败求助、高分炫耀、每日残局和互助充能都能自然分享 |

### 1.4.2 第一局 60 秒体验目标

首局不追求挑战，目标是让玩家完成“看见、理解、爽到、愿意再来一局”。

```text
0-5 秒：看到等距水晶浮岛和发光 Runner
5-15 秒：拖动唯一手牌，预览线自动接到终点
15-25 秒：松手后光流爆发，完成第一次通关
25-35 秒：结算三星和 Perfect Flow
35-45 秒：展示“你超过 82% 的新玩家”
45-60 秒：进入第 2 关，开始学习转弯
```

### 1.5 目标用户

- 微信小游戏轻中度用户。
- 喜欢解谜、路径规划、低压策略的玩家。
- 偏好精致视觉、治愈光效、短局体验的年轻及女性用户。
- 喜欢挑战高分、分享残局、群内互助的社交型玩家。

### 1.6 产品目标

| 指标 | 目标 |
| --- | --- |
| 单局时长 | 30-90 秒 |
| D1 留存 | 35%-45% |
| D7 留存 | 12%-18% |
| 新手首局完成率 | ≥ 90% |
| 前 5 关完成率 | ≥ 65% |
| 分享触发率 | ≥ 8% |
| 激励广告展示转化率 | ≥ 25% |
| 初版关卡 | 50 个主线关卡 + 1 个无尽模式 |
| 商业化结构 | 激励广告为主，轻度内购为辅 |
| 开发策略 | 14 天完成可验证 MVP，后续扩展关卡与主题 |

## 2. 核心玩法

### 2.1 核心循环

```text
进入关卡
  -> 生成起点、终点、障碍和初始道路
  -> 发放 3 张水晶手牌
  -> Runner 自动滑行或等待首次启动
  -> 玩家拖拽手牌到网格
  -> 系统实时预览路线
  -> 玩家放置、旋转、覆盖或弃牌
  -> Runner 沿道路前进
  -> 成功抵达终点 / 掉落死亡 / 触发 Near Miss
  -> 结算星级、光能、Perfect Flow、分享与成长
  -> 进入下一关或再次挑战
```

### 2.2 玩家操作

| 操作 | 说明 | 使用场景 |
| --- | --- | --- |
| 拖拽放置 | 将底部手牌拖入可放置网格 | 主操作 |
| 点击旋转 | 部分碎片放置后可切换方向 | 修正路线 |
| 长按手牌 | 触发子弹时间，降低全局速度 | 极限救场 |
| 覆盖道路 | 在 Runner 尚未到达的格子上替换已有 Tile | 动态修正 |
| 弃牌刷新 | 丢弃当前手牌并补发新牌 | 破除卡手 |
| 看广告救场 | 获得万能碎片、延时或补充子弹时间 | Near Miss 转化 |

### 2.2.1 易学规则压缩

对玩家展示的规则必须控制在三句话内：

1. 把水晶路块拖到发光小球前面。
2. 让光球沿着路走到终点。
3. 看到红色断线就换位置或旋转。

所有高级概念不在首屏解释，只在首次出现时用 1 个气泡提示：

| 高级概念 | 首次提示文案 | 出现时机 |
| --- | --- | --- |
| 旋转 | “点一下，让光拐弯。” | 第 2 关 |
| 预览 | “蓝线能通，红线会掉。” | 第 3 关 |
| 子弹时间 | “按住手牌，时间会变慢。” | 第 5 关 |
| 覆盖 | “还没走到的路，可以直接换。” | 第 7 关 |
| 颜色门 | “同色的光，才能通过同色的门。” | 第 21 关 |
| 分流 | “一束光会变成两束，两边都要安全。” | 第 31 关 |

### 2.3 游戏胜负

#### 胜利条件

- 关卡模式：Runner 或所有分裂 Runner 成功抵达终点。
- 多目标关：所有目标灯塔被对应颜色的光流点亮。
- 无尽模式：无固定胜利，目标是延长距离、连击与分数。

#### 失败条件

- Runner 前方无有效道路，滑入虚空。
- Runner 撞上障碍或不匹配的颜色门。
- 限时关卡时间耗尽。
- 分裂关卡中任意必要光流死亡。
- 手牌耗尽且无可用救场方式。

### 2.4 30 秒爽点模型

每一局都要在 30 秒内至少出现一次明确爽点，避免玩家只感到焦虑。

| 时间段 | 设计目标 | 反馈方式 |
| --- | --- | --- |
| 0-5 秒 | 给出清晰目标 | 起点、终点和推荐断点发光 |
| 5-15 秒 | 让玩家做出第一个有效决策 | 拖拽时实时出现蓝色预览线 |
| 15-25 秒 | 制造一次小危机 | Runner 接近断点、轻微慢镜头 |
| 25-30 秒 | 给予结果反馈 | 成功爆发 / Near Miss / 明确失败原因 |

### 2.5 策略深度来源

游戏不能只靠手速，进阶吸引力来自 4 层策略：

- 路线策略：选择更短、更安全或更高分的路线。
- 手牌策略：保留关键转弯牌，弃掉低价值牌，等待组合。
- 时间策略：决定何时使用子弹时间，何时冒险不减速。
- 风险策略：是否使用加速、分流、崩塌路线追求 Perfect 和高分。

## 3. 核心系统设计

### 3.1 双层网格系统

游戏表现为 2.5D 等距浮岛，底层使用 2D 矩阵逻辑。

- 默认地图尺寸：10 x 10。
- 坐标表示：`row, col`。
- 原点：左下角。
- 逻辑层：负责路线、碰撞、通关、障碍、手牌放置。
- 表现层：负责等距坐标、透视层级、动画、特效。

#### 坐标转换

```ts
screenX = (col - row) * (tileWidth / 2);
screenY = (col + row) * (tileHeight / 2);
```

### 3.2 Runner（水晶块）

Runner 是自动滑行的发光水晶生命体，也是玩家需要保护和引导的核心对象。

#### 状态机

| 状态 | 说明 |
| --- | --- |
| IDLE | 起点蓄力，等待教学或首次启动 |
| RUNNING | 正常滑行 |
| BULLET_TIME | 子弹时间中，整体减速 |
| SPLIT | 经过分流块后产生多个 Runner |
| SUCCESS | 抵达终点 |
| DEAD | 掉落、撞毁或规则失败 |

#### 关键属性

```ts
interface RunnerState {
  row: number;
  col: number;
  direction: Direction;
  color: 'none' | 'red' | 'blue' | 'yellow';
  speed: number;
  state: 'IDLE' | 'RUNNING' | 'BULLET_TIME' | 'SPLIT' | 'SUCCESS' | 'DEAD';
}
```

### 3.3 水晶道路碎片

每张手牌和每个地图道路块，本质都是一个方向映射模块。

```ts
type Direction = 'up' | 'down' | 'left' | 'right';
type RouteResult = Direction | 'dead' | Direction[];

interface TileConfig {
  id: string;
  type: TileType;
  routing: Partial<Record<Direction, RouteResult>>;
  rotatable?: boolean;
  oneTime?: boolean;
  colorFilter?: 'none' | 'red' | 'blue' | 'yellow';
  colorPaint?: 'none' | 'red' | 'blue' | 'yellow';
  speedModifier?: number;
}
```

### 3.4 碎片类型

| 类型 | 功能 | 策略定位 |
| --- | --- | --- |
| 直线水晶 | 沿当前方向直线通过 | 基础铺路 |
| 90 度折角镜 | 将 Runner 转向 90 度 | 基础转向 |
| 十字水晶 | 横竖方向均可通过 | 高容错铺垫 |
| T 字分流器 | 一路光流分裂为两路 | 多目标、高分、复杂关 |
| 偏振折射镜 | 特定进入方向产生特殊转向 | 高级解谜 |
| 加速晶体 | 经过后提高速度或分数倍率 | 风险收益 |
| 崩塌晶体 | Runner 离开后消失 | 限制回头与容错 |
| 染色晶体 | 改变 Runner 当前颜色 | 颜色机关前置 |
| 能量色门 | 仅允许对应颜色通过 | 核心解谜机制 |
| 万能水晶 | 可任意旋转并适配多数方向 | 广告救场奖励 |

### 3.5 覆盖与擦除规则

- 玩家可以覆盖 Runner 尚未到达的道路。
- Runner 正在经过的格子不可覆盖。
- 终点、起点、障碍、锁定机关不可覆盖。
- 覆盖后旧 Tile 直接消失，不返还手牌。
- 覆盖会立即刷新幽灵预览和路线预测。

### 3.6 子弹时间

子弹时间是本作的核心容错与操作爽点。

- 触发：长按任意手牌或拖拽手牌时自动进入。
- 效果：全局时间缩放至 0.3。
- 消耗：每关有独立能量条，默认可用 3 秒。
- 恢复：关卡内缓慢恢复，或通过广告立即补满。
- 视觉：背景降饱和，Runner 拖尾增强，屏幕边缘出现蓝色折射。

### 3.7 实时路径预览

当玩家拖拽手牌悬停在地图上方时，系统会在临时网格中放入该 Tile，并模拟 Runner 后续路线。

```ts
function previewRoute(grid: GridState, candidate: TilePlacement): PreviewResult {
  const tempGrid = grid.clone();
  tempGrid.place(candidate);
  return simulateRoute(tempGrid, grid.runnerStates);
}
```

预览表现：

- 可通向终点：显示高亮冰蓝光轨。
- 会死亡：显示红橙断裂虚线。
- 可触发 Perfect：显示粉紫闪光边缘。
- 分流路线：使用多条不同亮度光轨并行显示。

### 3.8 可控随机与保底机制

随机手牌要制造变化，而不是制造无解。

#### 发牌原则

- 每次发牌后，系统用 `RouteSimulator` 检查未来 3-5 步是否存在至少 1 条生路。
- 如果连续 2 次发出无效牌，第 3 次必须发出可连接当前断点的“救命牌”。
- 新机制首次出现的 3 关内，不允许同时出现卡手、限时和复杂障碍。
- 无尽模式中可以提高随机性，但必须保证玩家死亡前能明确知道自己哪里失误。

#### 手牌权重

| 玩家状态 | 发牌倾向 |
| --- | --- |
| 距离终点较远 | 提高直线、十字和基础折角概率 |
| 距离终点 1-3 格 | 提高可通向终点的关键牌概率 |
| 连续失败 2 次 | 增加万能水晶试用或提示 |
| 连续 Perfect | 提高加速、分流、崩塌等高风险牌概率 |
| 无尽模式高分段 | 降低保底权重，提高挑战性 |

### 3.9 提示与辅助系统

提示系统只解决理解问题，不直接代替玩家做决策。

| 辅助 | 触发 | 表现 |
| --- | --- | --- |
| 断点高亮 | Runner 前方 2 格无路 | 目标格轻微呼吸光 |
| 错误解释 | 放置后预览死亡 | 显示“这里会掉下去” |
| 方向箭头 | 新 Tile 首次出现 | Tile 表面显示进入和流出箭头 |
| 轻提示 | 同关失败 2 次 | 高亮一个推荐落点 |
| 强提示 | 同关失败 4 次 | 允许看广告获得一次完整路径影子 |

## 4. 关卡模式

### 4.1 主线关卡

主线关卡是首发核心内容，目标是让玩家逐步掌握路径构筑、转向、覆盖、颜色和分流。

#### 难度曲线

| 阶段 | 关卡 | 教学与挑战内容 |
| --- | --- | --- |
| 新手引导 | 1-5 | 直线、折角、拖拽、旋转、Perfect Flow |
| 基础压力 | 6-10 | Runner 自动滑行、限时、手牌刷新 |
| 机关入门 | 11-20 | 崩塌晶体、加速晶体、简单障碍 |
| 颜色解谜 | 21-30 | 染色晶体、能量门、颜色路线规划 |
| 分流策略 | 31-40 | T 字分流、多目标同时抵达 |
| 综合挑战 | 41-50 | 覆盖、子弹时间、Near Miss、复杂残局 |

### 4.1.1 进阶关卡挑战矩阵

进阶难度必须来自“组合复杂度”，而不是单纯加快 Runner。

| 维度 | 低阶 | 中阶 | 高阶 |
| --- | --- | --- | --- |
| 时间压力 | 无时间限制 | Runner 慢速前进 | Runner 加速 + 限时目标 |
| 空间限制 | 空网格较多 | 障碍形成绕路 | 可用格稀少且需要覆盖 |
| 手牌压力 | 固定手牌 | 受控随机 | 关键牌稀缺且需保留 |
| 机关复杂度 | 单一机关 | 两种机关组合 | 机关互相制约 |
| 路线目标 | 单终点 | 多灯塔 | 分流后同步抵达 |
| 评分目标 | 通关即可 | 限定步数 | Perfect + 连锁 + 不救场 |

### 4.1.2 关卡模板

首发关卡应围绕可复用模板生产，保证制作效率和难度稳定。

| 模板 | 核心乐趣 | 适用关卡 |
| --- | --- | --- |
| 断桥救场 | Runner 即将掉落，玩家补最后一块 | 5-15 |
| 绕障找路 | 障碍挡住直线，必须转弯绕开 | 6-25 |
| 最短路径 | 多条路可通，三星要求少用牌 | 10-50 |
| 颜色换轨 | 先染色再过对应门 | 21-35 |
| 双光同步 | 分流后两路都要安全抵达 | 31-50 |
| 崩塌回收 | 一次性道路要求不可回头 | 15-50 |
| 极限覆盖 | Runner 到达前替换错误道路 | 20-50 |
| 高分连锁 | 加速、分流、Perfect 组合追求分数 | 25-50 |

### 4.1.3 关卡失败后的再挑战设计

- 第 1 次失败：展示死亡原因，例如“前方没有路”“颜色不匹配”“分流少了一路”。
- 第 2 次失败：高亮一个可能的关键格，不直接给答案。
- 第 3 次失败：提供“看广告获得万能水晶”或“分享给好友帮我解”。
- 第 4 次失败：开放完整路径影子 3 秒，帮助玩家学习解法。
- 通关后：展示“如果少用 1 张牌可获得 Perfect”，引导二刷。

### 4.2 无尽模式

无尽模式强调高频操作和分享挑战。

- 没有固定终点。
- 地图随 Runner 前进向右上方滚动刷新。
- Runner 速度逐渐提高。
- 手牌随机性增强，但必须使用受控随机避免纯运气死亡。
- 每隔固定距离出现奖励浮岛或危险机关。
- 结算生成个性光轨海报，鼓励朋友圈和微信群分享。

### 4.3 残局挑战

残局挑战用于微信裂变。

- 来源 1：玩家失败时生成当前局面。
- 来源 2：玩家 Perfect 通关后生成高分复盘局面。
- 来源 3：系统每日生成官方残局。
- 分享方式：将关卡 ID、地图种子、Runner 状态、手牌状态压缩为分享参数。
- 群友点击后直接进入同一局面。
- 挑战成功后，分享者和挑战者都获得珍稀光能或抽奖券。

## 5. 新手体验（FTUE）

前 5 关必须零挫败，确保轻度用户在 60 秒内理解乐趣。

### 第 1 关：第一次连通

- 地图只保留起点、终点和 1 个高亮格。
- 手牌只有 1 张直线水晶。
- 玩家拖上去后自动发射光流。
- 目标：让玩家立即看到 Crystal Flow 爆发。

### 第 2 关：第一次转弯

- 引入 90 度折角。
- 不限时。
- 强提示旋转方向。

### 第 3 关：实时预览

- 玩家拖动时显示幽灵路线。
- 错误位置显示红色断裂线。
- 正确位置显示冰蓝光轨。

### 第 4 关：Perfect Flow

- 引入三星评分。
- 教玩家用更短路径获得 Perfect。

### 第 5 关：第一次救场

- Runner 开始自动滑行。
- 给出一次子弹时间机会。
- 让玩家体验“差点掉下去但救回来了”的爽感。

### 7.0.1 四类分享触发器

| --- | --- | --- | --- |
| 求助型 | Near Miss 失败 | “帮我补最后一块” | 让好友直接进入残局 |

### 7.1 残局挑战卡

玩家失败或高分通关时，可生成微信卡片。

推荐文案：

- “我只差 1 格就接通浮岛了，谁能帮我续上这束光？”
- “这条光轨超过了 95% 的玩家，全群谁能破？”

奖励：

- 分享者：群友挑战成功后获得复活、光能或稀有主题碎片。

### 7.1.1 残局卡内容规范

残局卡必须让未玩过的人也能一眼理解。

| 元素 | 规范 |
| --- | --- |
| 标题 | 只强调一个问题，例如“差 1 格通关” |
| 图片 | 显示起点、终点、断点和 3 张手牌 |
| 按钮 | “帮 TA 接上”或“挑战这条光轨” |
| 难度 | 用 1-5 个水晶图标表示 |
| 奖励 | 明确写出“成功双方 +50 光能” |
| 入口 | 点击后不进主界面，直接进残局 |

### 7.2 互助充能

- 每局消耗 1 点精力。
- 精力不足时，可看广告或请求好友充能。
- 好友点击后帮助玩家恢复 1 点精力。
- 好友自身获得光能或抽奖券。

### 7.2.1 互助充能限制

- 每日被助力上限：5 次。
- 每位好友每日只能助力同一玩家 1 次。
- 新用户首次助力后直接获得 3 局免精力体验。
- 不能在新手前 5 关卡体力，避免过早打断学习。

### 7.3 个性光轨海报

无尽模式或 Perfect 通关后生成一张纯视觉海报。

画面内容：

- 深色渐变背景。
- 本局完整发光路径。
- 主题色粒子。
- 玩家称号。
- 路径风格文案。

示例文案：

- 直线型路径：“一往无前的光，也需要有人铺路。”
- 曲折型路径：“兜兜转转，最后还是接住了这束光。”
- 分流型路径：“真正的答案，不止一条路。”

### 7.4 群排行榜与每日残局

- 每日 12:00 自动刷新 1 个官方残局。
- 群内玩家共享同一排行榜，比较用牌数、完成时间和 Perfect。
- 排行榜只显示好友和同群玩家，降低陌生竞争压力。
- 每日首次分享官方残局不消耗精力。
- 群排行榜结算后给前 3 名发放头像框或光能，不发放破坏平衡的强道具。

### 7.5 防骚扰规则

- 任意分享入口必须允许关闭，不能强制分享。
- 同一局失败只弹出 1 次分享求助。
- 玩家连续拒绝分享 3 次后，当日降低分享弹窗频率。
- 分享奖励以视觉、光能、体力为主，不给付费货币大额奖励。

## 8. 商业化方案

### 8.1 变现原则

- 不在新手前 5 关强插广告。
- 广告必须服务于玩家明确痛点。
- 广告奖励必须即时、可感知、与失败挽回相关。
- 不让付费破坏解谜核心，只提供便利和视觉定制。

### 8.2 激励广告点位

| 点位 | 触发场景 | 奖励 | 优先级 |
| --- | --- | --- | --- |
| Near Miss 拯救 | 差 1-2 格通关或即将坠落 | 万能水晶 / 延长 30 秒 / 复活 | 高 |
| 手牌刷新 | 当前 3 张牌无法破局 | 刷新全部手牌 | 高 |
| 子弹时间补能 | 能量条耗尽 | 立即补满 | 中高 |
| 第四手牌槽 | 卡手频繁或无尽模式 | 解锁 24 小时备用槽 | 中高 |
| 结算倍增 | 通关或 Perfect 后 | 光能 x3 | 中 |
| 主题试玩 | 新主题首次曝光 | 限时解锁色彩包 | 中 |

### 8.2.1 广告触发策略

广告必须出现在“玩家已经产生需求”的瞬间。

| 玩家痛点 | 最佳广告话术 | 奖励设计 |
| --- | --- | --- |
| 差一点通关 | “补上最后一块，继续这束光” | 获得 1 张万能水晶 |
| 卡手无解 | “重洗手牌，换一组可能性” | 刷新当前 3 张手牌 |
| 操作来不及 | “让时间慢下来” | 子弹时间补满 |
| 奖励不够 | “让这次 Perfect 更值钱” | 光能 x3 |
| 主题想试 | “这局用暮色罗兰跑一次” | 主题试用 30 分钟 |

### 8.2.2 广告频控

- 新手前 5 关不主动弹广告。
- 每局最多展示 1 次 Near Miss 拯救广告入口。
- 同一失败局最多展示 2 个广告选择，避免选择过载。
- 玩家看广告后必须立刻回到局内并使用奖励，不允许二次确认。
- 连续 2 次广告加载失败时，免费给一次低配救场，防止流失。
- 每日广告观看达到 8 次后，降低额外弹窗，只保留主动入口。

### 8.2.3 广告 A/B 测试指标

| 测试项 | A 方案 | B 方案 | 核心指标 |
| --- | --- | --- | --- |
| Near Miss 奖励 | 万能水晶 | 延长 30 秒 | 广告完成率、救场成功率 |
| 结算倍增 | 光能 x2 | 光能 x3 | 展示转化率、次日留存 |
| 第四槽时长 | 1 小时 | 24 小时 | 广告观看次数、无尽模式时长 |
| 分享替代广告 | 看广告复活 | 好友助力复活 | 分享率、回流率、收入 |

### 8.3 轻度内购

- 无广告月卡：无需观看广告即可领取救场、刷新和补能奖励。
- 视觉主题包：极光冰原、暮色罗兰、落日余晖、黑曜石、赛博霓虹。
- 光流特效包：金色流光、星尘尾迹、岩浆裂纹、冰晶爆闪。
- 章节包：后续扩展高难关卡章节。

### 8.4 变现漏斗

```text
第 1-5 关：只建立理解和爽感，不变现
第 6-10 关：出现第一次可选结算倍增
第 11-20 关：出现 Near Miss 拯救和手牌刷新
第 21-30 关：颜色门提高广告救场价值
第 31 关后：分流、多目标和无尽模式支撑第四槽、子弹时间补能、主题售卖
```

### 8.5 付费与公平边界

- 付费和广告不能直接跳过主线关卡。
- 万能水晶只解决当前一步，不自动完成整关。
- 无广告月卡只是免看广告领奖励，不提高奖励倍率。
- 皮肤、主题和尾迹不影响路线判定。
- 排行榜需标记使用救场次数，Perfect 榜只允许无救场成绩进入。

### 12.2 路线模拟伪代码

```ts
function simulateRoute(grid: GridState, runners: RunnerState[]): PreviewResult {
  const queue = [...runners];
  const paths: PathNode[][] = [];
  const maxSteps = grid.rows * grid.cols * 2;

  while (queue.length > 0) {
    const runner = queue.shift()!;
    const path: PathNode[] = [];
    let current = { row: runner.row, col: runner.col };
    let direction = runner.direction;
    let color = runner.color;

    for (let step = 0; step < maxSteps; step++) {
      const next = move(current, direction);
      if (!grid.isValid(next)) {
        path.push({ ...next, result: 'dead' });
        break;
      }

      const tile = grid.getTile(next);
      if (!tile) {
        path.push({ ...next, result: 'dead' });
        break;
      }

      if (tile.colorFilter && tile.colorFilter !== 'none' && tile.colorFilter !== color) {
        path.push({ ...next, result: 'blocked' });
        break;
      }

      if (tile.colorPaint && tile.colorPaint !== 'none') {
        color = tile.colorPaint;
      }

      const route = tile.routing[direction];
      if (!route || route === 'dead') {
        path.push({ ...next, result: 'dead' });
        break;
      }

      path.push({ ...next, result: 'pass' });

      if (Array.isArray(route)) {
        for (const splitDirection of route) {
          queue.push({ ...runner, row: next.row, col: next.col, direction: splitDirection, color });
        }
        break;
      }

      current = next;
      direction = route;
    }

    paths.push(path);
  }

  return evaluatePreview(paths, grid.goals);
}
```

## 13. 关键风险与解决方案

| 风险 | 表现 | 解决方案 |

| 随机挫败 | 手牌连续无效导致失败 | 受控随机、保底牌、广告刷新 |
| 规则难懂 | 用户不知道光路如何走 | 前 5 关强引导、幽灵预览、箭头提示 |
| 视觉廉价 | 零美术导致画面粗糙 | 统一材质、粒子、渐变、等距层级 |
| 性能压力 | 粒子和透明材质过多 | 限制粒子数量、对象池、低端机降级 |
| 分享骚扰 | 玩家反感强制裂变 | 分享只出现在失败救场、高分炫耀和互助充能 |
| 广告打断 | 影响短局节奏 | 只做激励广告，不做强插屏 |
| 玩法单调 | 只铺路容易疲劳 | 颜色、分流、崩塌、无尽、残局轮换 |

## 14. 最终产品定位

《浮岛浮光》不是传统拼图，也不是传统跑酷，而是一款“玩家不直接控制角色，而是实时构筑世界规则”的轻策略小游戏。

最终定位：

> 一款用等距水晶美术包装、以动态路径构筑为核心、适合微信生态裂变和短局上瘾体验的逆向跑酷解谜游戏。

核心竞争力：

- 低成本：无需大量原画资产。
- 高辨识：水晶、光轨、等距浮岛形成强视觉记忆点。
- 易传播：残局挑战和光轨海报天然适合微信群与朋友圈。
- 可扩展：Tile 规则、颜色、分流、主题和关卡都能持续追加。
- 可验证：14 天内可以完成 MVP 并进行真实用户测试。
